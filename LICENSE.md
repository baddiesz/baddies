## 🐛 Common Issues and Fixes

### ❌ GUI doesn't appear after execution
**Fix:** The panel may be off-screen or hidden behind the Roblox UI. Try
resetting your character or rejoining the server and executing again.

### ❌ Auto stomp stops triggering after a few minutes
**Fix:** Some Baddies updates reset the stomp detection hook. Update to
the latest script version — v2.3.1 includes a persistent re-hook loop.

### ❌ Kill all only eliminates some enemies
**Fix:** Enemies that spawned after script execution won't be in the
initial target list. Enable the **auto kill loop** instead — it
continuously catches new spawns in real time.

### ❌ All weapons unlock doesn't apply in-game
**Fix:** Execute the script before picking up any weapon. The unlock
module needs to initialize before the game's weapon system loads fully.

### ❌ Script crashes on Android executor
**Fix:** Switch to **Arceus X** or **Delta** — these are the two most
tested executors for the mobile build. Make sure the executor itself
is updated to its latest version.

### ❌ Teleport sends character out of bounds
**Fix:** Use only the preset teleport locations in the GUI dropdown.
Manual coordinate inputs require precise Y-axis values to avoid
spawning underground.

### ❌ "HttpGet failed" error on load
**Fix:** Verify your internet connection and confirm that your executor
has HTTP request permissions enabled in its settings panel.
